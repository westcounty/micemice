# Micemice V1 Sprint 计划与DOD

- 版本：V1.0
- 日期：2026-02-07

## 1. Sprint节奏

- 每个Sprint 2周，总计4个Sprint。
- 需求冻结：每个Sprint开始前2天。
- 演示与复盘：每个Sprint最后1天。

## 2. Sprint目标

## Sprint 1（基础设施与权限）

1. 完成项目分层脚手架。
2. 完成登录、会话、RBAC。
3. 完成审计日志基础能力。
4. 完成主数据模块最小闭环（品系、房间、机架、笼位）。

交付物：

1. 可运行App骨架。
2. 基础API联调通过。
3. 权限矩阵文档。

## Sprint 2（笼位与个体）

1. 笼位列表与笼卡详情。
2. 扫码进入笼卡。
3. 个体列表与详情。
4. 转笼/并笼/拆笼流程。

交付物：

1. 现场操作主流程可演示。
2. 超笼预警可配置。

## Sprint 3（繁育与分型）

1. 配种向导、查栓、产仔、断奶。
2. 采样登记、批次管理、分型回填。
3. 任务中心自动生成规则。

交付物：

1. 繁育到分型端到端链路。
2. 冲突分型人工确认流程。

## Sprint 4（Cohort/报表/导出）

1. cohort筛选与锁定。
2. 基础报表与导出。
3. 合规导出包。
4. 性能稳定性收敛与发布准备。

交付物：

1. 可验收的V1版本。
2. 用户手册与运维说明。

## 3. Definition of Done（统一）

1. 需求验收标准全部满足。
2. 代码通过lint与单测。
3. 关键路径有自动化测试覆盖。
4. 审计日志埋点完成。
5. 文档同步更新（接口、字段、流程）。
6. QA通过并关闭阻断缺陷。

## 4. 发布门禁

1. Sev1缺陷为0。
2. Sev2缺陷<=2且有替代方案。
3. 关键性能达标。
4. 合规阻断缺陷为0。

## 5. 风险前置检查

1. 字典是否冻结。
2. 规则是否与生物学专家确认。
3. 数据迁移模板是否验证。
4. 现场网络不稳定场景是否演练。
