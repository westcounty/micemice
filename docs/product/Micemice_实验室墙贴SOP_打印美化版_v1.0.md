# Micemice 实验室现场执行规范
## Wall SOP · 打印美化版 v1.0（建议 A4 纵向，黑白打印）

---

## A. 今日优先级（先后顺序不可变）

1. `合规风险`：协议到期、无资质、关键阻断
2. `动物福利`：超笼、异常个体、逾期操作
3. `实验节点`：配种、断奶、采样、实验事件

---

## B. 标准作业流程（SOP）

### B1. 开工前（2 分钟）

1. 登录并确认角色
2. 查看工作台与通知中心
3. 记录今日第一优先任务

### B2. 现场执行（每笼循环）

1. 扫码进笼卡（失败则手动输入）
2. 执行笼位动作（转笼/并笼/拆笼/批量状态）
3. 进入个体详情补充记录：
   1. 事件：体重/健康/处理
   2. 附件：文件选择或拍照直传
4. 回任务页勾选完成

### B3. 关键流程顺序

1. 繁育：`配种 -> 查栓 -> 产仔 -> 断奶向导`
2. 分型：`采样 -> 建批 -> 导入 -> 冲突确认`
3. 实验：`建实验 -> 事件记录 -> 归档`

---

## C. 角色职责卡

1. 实验员（Researcher）：
   1. 执行流程
   2. 保证记录完整
2. PI：
   1. 看报表趋势
   2. 处理异常决策
3. 管理员（Admin）：
   1. 主数据维护
   2. 权限矩阵（RBAC）
   3. 审计与导出备份

---

## D. 红线与强制规则

1. 协议无效/过期：关键动作不可执行
2. 状态机非法跳转：一律禁止
3. 缺权限操作：不得绕过，必须走权限开通
4. 关键动作必须可审计（含 Before/After）

---

## E. 1 分钟异常处理

1. 扫码失败 -> 手输笼号 -> 重打二维码
2. 提示缺权限 -> 管理员调整权限矩阵
3. 会话退出 -> 重新登录（30 分钟超时机制）
4. 导出失败 -> 重新导出并核对路径

---

## F. 收工检查（班次结束前）

1. 关键任务状态确认
2. 通知中心清理（已读/处置）
3. 导出文件备份（至少 CSV）
4. 异常记录已补充说明

---

## G. 现场执行口号

`先合规，后效率；先记录，后离场。`
