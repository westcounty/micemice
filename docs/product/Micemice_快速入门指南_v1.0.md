# Micemice 快速入门指南 v1.0

- 版本：v1.0
- 日期：2026-02-07
- 适用对象：第一次使用 Micemice 的实验室成员

## 1. 5 分钟了解

Micemice 是一个离线优先的实验室小鼠管理 App。  
你可以在一个端里完成：
1. 笼位与个体管理
2. 繁育与分型流程
3. Cohort 与实验管理
4. 任务、通知、审计、导出
5. 管理端权限矩阵与主数据治理

## 2. 首次启动

1. 安装并打开 App（Debug 包常见路径：`app/build/outputs/apk/debug/app-debug.apk`）。
2. 登录页输入：
   1. 账号：任意非空
   2. 密码：至少 4 位
   3. 组织编码：非空（可用 `LAB-A1`）
3. 选择登录角色：
   1. 博士生/实验员
   2. PI
   3. 管理员
4. 点击“进入系统”。

## 3. 首日推荐流程（管理员先做）

## Step 1：基础治理（约 8 分钟）

在“管理”页完成：
1. 主数据：
   1. 品系字典
   2. 基因型模板
2. 协议状态：
   1. 启用当前有效协议
   2. 确认即将到期协议提醒天数
3. 培训资质：
   1. 为操作者账号配置有效期
4. 通知策略：
   1. 协议到期
   2. 任务逾期
   3. 容量预警
   4. 同步失败
5. 权限矩阵：
   1. 确认各角色权限点符合实验室制度

## Step 2：准备资源（约 5 分钟）

在“管理”页继续：
1. 创建笼位（房间/机架/位点/容量）
2. 创建个体（耳号/RFID、品系、基因型、入笼）
3. 生成笼码二维码并导出 PNG（建议打印贴笼）

## Step 3：执行端演练（约 12 分钟）

切换到“博士生/实验员”角色：
1. 扫码进入笼卡
2. 尝试转笼/并笼/拆笼或批量状态更新
3. 进入个体详情，记录：
   1. 体重/健康/处理事件
   2. 附件（选择文件或拍照直传）
4. 繁育流程演练：
   1. 创建配种计划
   2. 查栓登记
   3. 产仔登记
   4. 断奶向导
5. 分型流程演练：
   1. 采样登记
   2. 建批
   3. 导入结果
   4. 冲突确认

## Step 4：PI 复盘（约 5 分钟）

切换到“PI”角色：
1. 打开报表页，查看：
   1. 占用率/完成率/繁育成功率/存活率
   2. 品系分布与任务状态分布
   3. 任务完成率趋势
2. 在工作台确认关键风险项是否收敛。

## 4. 一键可用的演练脚本（建议照做）

1. 管理员登录，新增一个笼位 `C-901`（容量 5）。
2. 创建两只亲本个体（1 雄 1 雌）并入笼。
3. 生成 `C-901` 二维码并保存。
4. 切到实验员：
   1. 扫码进入 `C-901`
   2. 创建一个配种计划
   3. 给雌鼠添加一次体重事件（如 22.3g）
   4. 给个体添加一张拍照附件
5. 切到 PI：
   1. 报表页看趋势变化
   2. 核对任务完成率是否更新
6. 切回管理员：
   1. 导出合规 CSV/PDF/ZIP
   2. 审计页确认有对应 Before/After 记录

## 5. 角色切换与权限排障

1. 看不到“管理”页：当前角色不是 Admin。
2. 操作时报“缺少权限”：被权限矩阵禁用。
3. 处理方法：
   1. Admin 登录
   2. 进入“细粒度权限矩阵”
   3. 按角色开启对应权限点

## 6. 数据与导出建议

1. 每周至少导出一次：
   1. 个体 CSV
   2. 合规 CSV/PDF
   3. 合规 ZIP 包
2. 检查导出文件路径并同步到实验室备份盘。
3. 审计日志建议长期保留。

## 7. 常见问题（快速解答）

1. 登录失败  
检查账号/组织是否为空，密码是否少于 4 位。

2. 任务看起来“消失”  
多为筛选条件导致，切换到“全部”查看。

3. 为什么会自动退出  
30 分钟无操作会话超时，属安全机制。

4. 扫码失败怎么办  
先手工输入笼号兜底，再检查二维码打印清晰度。

## 8. 上线前检查清单（建议）

1. 协议状态全部核对完毕
2. 培训资质已覆盖实际操作者
3. 权限矩阵通过 PI/管理员确认
4. 笼码已打印并贴标
5. 导出链路（CSV/PDF/ZIP）验证通过
6. 关键流程（配种-断奶-分型-实验）全链路演练通过
